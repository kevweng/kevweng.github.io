The importance of setting up your work environment

Sep 7, 2018 3 min read

A lot of people begin their journeys in learning programming with asking what environment they will be coding in. To a lot of people—and hopefully you as well after you read this post—the digital workspace will have a newfound meaning.

I used to think that integrated development environments (IDEs) were the end all be all of professional software development tools. I looked at the very minute differences in debugging services that IDEs like CLion or PyCharm had in comparison to Eclipse or some other IDE. This was due to the relatively high prices of getting the software. Eventually, I found out that students were able to license the software for free and I downloaded the few JetBrains IDEs that I wanted. They went unopened. In retrospect, I think that this was too easy, and I eventually abandoned the idea of using a heavy IDE that took a minute to boot.

I want to make it clear that I don’t think IDEs are bad products, the point of this post is to say that the overload of tools that comes wrapped in a package was definitely discouraging to me, and I suggest learning to program from the shell.

There are many benefits to having this skill such as writing to a server directly.

Since then, I have discovered the power of the terminal. For Windows users, the shell most are familiar with is the command prompt, but I use MacOS. I have since learned vim, and find that I am more impressed everyday with its brute power, yet minimal bloat over a traditional IDE. It’s lightweight and comes with the terminal. For my terminal however, I decided to switch to iTerm2, which is a shell emulator that runs instead of the built in MacOS Terminal. I have found that the customization process, in both vim and iTerm2 has been a motivation to continue programming. It’s not difficult to get lost in all of the different color schemes, key mappings, and other appearance tweaks that you can make to your development environment.

I encourage you take the time to make it yours. You want your work environment to be as comfortable and relatable as possible so that when you’re doing boring work, at least you’re doing it in a pretty environment. For me, I followed guides on how to configure my terminal to have clear arrows showing directory paths and git branches. There’s syntax highlighting to make things more readable, and timestamps with numbered lines of code as well as error markings. These cosmetic differences make me no better of a programmer than someone that just uses the base terminal. However, I think that the process in tweaking various config files is very useful in navigating the terminal. A lot of people want to get into programming to learn it as a skill and lack a project or task in mind that they want to accomplish. I think that customizing the digital environment is a great first task for learning, and most importantly, employing the navigation commands. You learn so much from troubleshooting small issues that arise in the process. Google heavily, and you’ll find that in every problem you run into as a beginner, there are ten more people who’ve been there before you. People know that you need to learn through experience, but are missing that first task to guide them through the environment they’re working with and teach them the tools they will be using. As a bonus, you’ll look like a total badass when your terminal is all colorful. Make sure you use a dark theme; your eyes will thank you later.
